What is responsive design?

Responsive Design might better be named "Device-Independent Design" or "Flexible-Width Design." It is the term commonly used to describe a web layout that adjusts itself (or "responds") to the available width of the window in which it is being viewed — from small mobile device screens to 4k displays.

Responsive layout is an expected feature of any modern web design, and is perhaps the biggest single factor that makes designing for screens different than designing for print or other media. The good news is that browser default styles are (for the most part) already responsive. That is, an HTML document with no stylesheet will generally display without complaint on any device's browser, and the only user-interaction required to view the content will be vertical scrolling. (Images are a notable exception to this statement, but one that is easy to remedy with a simple line of CSS: img { max-width: 100%; })

The viewport meta-tag

In the early days of the web, designers built fixed-width layouts for a 'lowest common denominator' screen size, usually 960px wide. That quickly became untenable with the advent of smartphones and high-resolution displays. Smaller devices needed a way to handle these 'legacy' pages, and a way to distinguish them from responsive ones. Enter the viewport meta tag. This tag tells the browser that our page uses responsive layout, and should not apply the browser’s default 'pinch and zoom' feature intended for old fixed-width pages.

The tag usually looks like this: meta name="viewport" content="width=device-width, initial-scale=1.0". It belongs in the head of the document and is critical to proper rendering of your page on a mobile device.

Testing responsive behavior

Use Chrome DevTools to emulate different devices and test how your page reflows. The “Toggle Device Toolbar” button lets you simulate phones, tablets, or any custom width.

You can then adjust widths and see how the page responds dynamically.

Manage scale by setting limits, rather than fixed widths

Instead of fixed widths, use relative units and limits like width: 90% and max-width: 40rem. This keeps text comfortable to read across all devices.

Responsive images

To ensure images scale correctly, use: img { max-width: 100%; }. This prevents images from overflowing their containers.

Media queries

Media queries let you apply different styles depending on screen size. For example:

html { font-size: 12pt; }

@media (min-width: 640px) {
  html { font-size: 16pt; }
}

When the screen width reaches 640px, the font size increases for better readability.
